# mojo.nvim Design

**Status:** Approved

## Goal

Create a generic Neovim plugin that centralizes Mojo editor integration while keeping official Mojo tools external.

## Architecture

- `lua/mojo/init.lua` is the public entrypoint.
- `lua/mojo/config.lua` owns defaults and merge logic.
- `lua/mojo/filetype.lua` registers `.mojo` and `🔥` extension detection.
- `lua/mojo/env.lua` detects Pixi and venv project environments.
- `lua/mojo/treesitter.lua` registers the Mojo parser entry and keeps it isolated for future replacement.
- `lua/mojo/lsp.lua` and `lua/mojo/format.lua` expose adapters for external configuration managers.
- `lua/mojo/adapters/lazyvim.lua` provides opt-in helpers for LazyVim.

## Rules

- Do not bundle the Mojo LSP binary.
- Prefer official Mojo tools when available.
- Keep custom logic small, documented, and replaceable.
- Make Treesitter swappable without changing the public API.
